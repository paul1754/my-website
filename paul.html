7. Application Layer

The application layer is the topmost layer of the OSI model and the one that we interact with directly as users. It handles things like web browsers, email programs, and file transfers. This layer provides protocols that help send, receive, and display data in a way that makes sense to people.

Some common examples of protocols here are:

  • Hypertext Transfer Protocol (HTTP) – used for loading websites
  • File Transfer Protocol (FTP) – for uploading/downloading files
  • Simple Mail Transfer Protocol (SMTP) – for sending emails
  • Post Office Protocol (POP) – for receiving emails
  • Domain Name System (DNS) – turns domain names like google.com into IP addresses

6. Presentation Layer

The presentation layer (also called the syntax layer) makes sure data from the application layer is in a format that can be understood by the receiving device. It handles things like translating between different character encodings (e.g., ASCII to EBCDIC), compressing files to save bandwidth, and encrypting data for security.

Basically, it takes the raw data from the application layer, prepares it properly (encryption, compression, formatting), and passes it down to the session layer for transmission.

5. Session Layer

The session layer manages the communication sessions between two devices. It sets up the connection (like "opening a session"), keeps it going smoothly while data is flowing, and closes it cleanly when done.

It can also create checkpoints (sync points) during long transfers. If the connection drops (e.g., network glitch), the devices can pick up from the last checkpoint instead of starting over. 6. Presentation Layer

The presentation layer prepares data for the application layer. It defines how two devices should encode, encrypt, and compress data so it is received correctly on the other end. The presentation layer takes any data transmitted by the application layer and prepares it for transmission over the session layer.



5. Session Layer

The session layer creates communication channels, called sessions, between devices. It is responsible for opening sessions, ensuring they remain open and functional while data is being transferred, and closing them when communication ends. The session layer can also set checkpoints during a data transfer—if the session is interrupted, devices can resume data transfer from the last checkpoint.
